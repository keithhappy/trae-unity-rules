# Trae Unity Rules

当检测到当前工作区为 Unity 工程时，加载规则文件并严格遵循。

## 触发条件

满足以下条件时生效：

1. 当前工作区包含 `Assets/`、`ProjectSettings/` 等 Unity 工程标志性目录或文件。
2. 工程根目录下存在 `.trae/rules/` 目录，且其中包含 `.md` 文件。
   - 若无 `.trae/rules/` 或其中无 `.md` 文件：回退使用本 Skill `references/` 目录下的默认规则。

## 工作流程

### 1. 检测 Unity 工程

检查当前工作区根目录是否包含以下任一标志：
- `Assets/` 目录
- `ProjectSettings/` 目录
- `Packages/manifest.json`
- `.unity` 文件

### 2. 加载规则文件（合并策略）

**规则来源：**
- **默认规则**：本 Skill `references/` 目录下的 `.md` 文件（随 Skill 分发，作为兜底基准）。
- **工程规则**：当前工程 `.trae/rules/` 目录下的 `.md` 文件（工程定制，优先级高于默认规则）。

**合并逻辑：**
- 以默认规则为基线。
- `.trae/rules/` 中存在的 `.md` 文件：覆盖同名的默认规则（工程规则优先）。
- `.trae/rules/` 中存在但默认规则中没有的 `.md` 文件：视为工程新增规则，合并到最终规则集中。
- 最终使用的规则 = 默认规则（未被覆盖的部分） + 工程规则（全部）。
- 读取所有最终规则文件的完整内容，不可只读摘要或部分内容。

### 3. 遵循规则

- 在编写、修改、审查任何 C# 代码时，必须严格遵循最终合并后的规则。
- 规则文件的优先级高于默认编码风格。
- 如果不同规则文件之间有冲突，以文件名排序靠前的为准。
- 如果规则与用户显式指令冲突，以用户指令为准，但应提醒用户差异。

## 注意事项

- 每次对话开始时都应重新检查规则文件是否更新（可通过 `.rules-version` 文件判断版本变化）。
- 不修改 `.trae/rules/` 目录及 `references/` 目录中的任何文件。
